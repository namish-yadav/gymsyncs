# 💡 Why GymSync?

## Why did I build this project?

As I began learning web development, I noticed that many beginners—including myself—often build small tutorial projects, complete them, and never use them again.

I wanted to take a different approach.

Instead of creating dozens of disconnected projects, I decided to build one application that I would genuinely use every day and continuously improve as I learned new technologies.

GymSync is that project.

---

# 🎯 Problem Statement

Many people struggle to stay consistent in the gym because they don't keep accurate records of their workouts.

Common problems include:

- Forgetting previous workout weights
- Losing track of progressive overload
- Forgetting exercise order
- Missing workout history
- Losing motivation due to lack of measurable progress

Although many fitness apps already exist, many include unnecessary complexity, subscriptions, or features that I personally didn't need.

GymSync focuses on creating a clean, fast, and distraction-free experience.

---

# 🛠️ Why HTML, CSS & JavaScript?

Version 1 intentionally uses only HTML, CSS and Vanilla JavaScript.

I wanted to master web fundamentals before introducing frameworks.

Learning the fundamentals helps me better understand:

- Semantic HTML
- Responsive Design
- CSS Layout
- JavaScript Logic
- DOM Manipulation
- Browser APIs
- Local Storage

Once I become comfortable with these concepts, I plan to migrate the project to React.

---

# 💾 Why Local Storage?

The first version is designed to work completely offline.

Benefits include:

- No backend required
- Faster development
- Easier debugging
- No internet connection required
- Simple deployment

As I learn backend development, Local Storage will eventually be replaced with a real database and cloud synchronization.

---

# 🚀 Why not React?

Although React is widely used in the industry, I wanted to avoid relying on frameworks before understanding core JavaScript.

Building Version 1 with Vanilla JavaScript strengthens my fundamentals and makes it easier to appreciate the benefits of React when I migrate later.

---

# 🧠 Why one long-term project?

Many portfolios contain dozens of unrelated tutorial projects.

I believe one continuously evolving application better demonstrates:

- Long-term commitment
- Planning
- Problem solving
- Software architecture
- Version control
- Continuous improvement

GymSync will grow with my skills throughout my BCA, making it both a practical tool and a record of my progress as a developer.
